# サマリー形式ガイド

## 基本構造

```
【発表】何が発表/公開されたか（1行）
【意義】なぜ重要か、何が変わるか（1行）
【ポイント】
・主要ポイント1
・主要ポイント2
・主要ポイント3（任意）
```

## 文字数

- 全体: 150-250文字
- 各ポイント: 20-40文字

## 良い例

```
【発表】Gemma Scope 2 を公開
【意義】AI安全性研究者がモデル内部動作を可視化可能に
【ポイント】
・Sparse Autoencoders で特徴を自動抽出
・27B パラメータモデルまで対応
・オープンソースで即利用可能
```

## 悪い例

```
Gemma Scope 2を公開し、AI安全性コミュニティが複雑な言語モデルの挙動を深く理解できるよう支援。
```
→ 具体性がなく、何ができるか不明

## カテゴリ別の注意点

| カテゴリ | 重視するポイント |
|----------|------------------|
| Model | モデル名、パラメータ数、性能向上率、用途 |
| Service | 機能名、対象ユーザー、料金影響、利用開始時期 |
| Other | 関係者、規模、影響範囲 |
